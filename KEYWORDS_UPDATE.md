# Обновления вкладки "Keywords" в настройках

## Последние изменения (2024-12-19)

### Крестик показывается только при наличии текста
- **Проблема**: Крестик был всегда виден, даже когда поле ввода пустое
- **Решение**: Добавлена логика показа/скрытия крестика в зависимости от наличия текста
- **Реализация**: 
  - CSS: крестик скрыт по умолчанию (`opacity: 0; visibility: hidden`)
  - JavaScript: добавлен класс `has-text` к контейнеру при наличии текста
  - Крестик показывается при вводе, фокусе и скрывается при очистке поля

### Счетчик в контрастных цветах
- **Проблема**: Цвета полосок счетчика были недостаточно контрастными к синему фону
- **Решение**: Изменены цвета на более контрастные
- **Новые цвета**:
  - Полоски: `#FFD93D` (желтый) вместо зеленого
  - Счетчик: `#FF6B6B` (красный) с черной рамкой и тенью
  - Добавлены тени и улучшена читаемость

### Технические детали
- **CSS**: Обновлены стили для `.keywords-clear-button` и `.pixel-counter`
- **JavaScript**: Добавлена функция `updateKeywordsInputState()` для управления состоянием крестика
- **Обработчики**: Добавлены event listeners для `input`, `focus` и очистки поля
- **Автоматическое обновление**: Состояние крестика обновляется при всех изменениях поля ввода

## CSS классы
- `.keywords-window-header` - заголовок окна с синим фоном
- `.keywords-window-title` - заголовок "КЛЮЧЕВЫЕ СЛОВА"
- `.pixel-counter` - контейнер счетчика в пиксель-стиле
- `.pixel-bar` - декоративные полоски по бокам счетчика
- `.favorites-count` - сам счетчик с красным фоном
- `.keywords-input-container.has-text` - состояние при наличии текста (показывает крестик)
- `.keywords-clear-button` - кнопка очистки (скрыта по умолчанию)

## JavaScript функции
- `updateKeywordsInputState()` - обновляет состояние крестика
- `addKeyword(keyword)` - добавляет ключевое слово
- `removeKeyword(keyword)` - удаляет ключевое слово
- `updateKeywordsInDatabase()` - сохраняет в Supabase
- `loadKeywords()` - загружает из Supabase
